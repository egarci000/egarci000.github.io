Professional Self Assessment

Developing this ePortfolio gave me great insight into my abilities in programming and crafting a narrative on my skills as a programmer. The ePortfolio includes a simple web interface I built, where users can load school courses from a CSV file and display them on the page. They can edit courses, delete one or all courses, and export the courses to another CSV file. The program uses a MongoDB database to save the courses while the user’s session is active. It also gives them an option to connect to a database on their own local computer. In this way, my skills in working with databases are shown.

I also created a starter command line interface app from where the web interface was created. Furthermore, I also built a mongo_crud file with the basic database CRUD operations like create, read, update and delete. Using this file, I was able to do the required database operations in both the web interface app and the much simpler CLI app.

By working on this application, I was able to sharpen my skills in data structures and algorithms. I was able to design an AVL tree in the CLI app that would take the courses a user loads from a CSV file and load them in the program temporarily. This AVL tree replaced the simpler binary tree that was originally implemented. This new data structure was able to load courses much faster and thus shows my ability to think of programs not only in their function but their speed as well, a category that can never be overlooked. I was also able to focus on app security concepts like proper error handling. Where appropriate, I designed error cases where if the program runs into an error, it does not return a default error message which could be exploited by bad actors, but a custom error message. This not only protects the app backend but also prevents the web from crashing, as the app will display the error message to the user but does not cause it to stop running.

My artifact deals with important categories in computer science like software engineering and design, algorithms and data structures, and databases. I was able to design a fully functioning CLI app that allowed users to load courses from a CSV file, save them to a database, and print out course information. I was then able to take that design and build a fully functioning web app that allowed users to perform the same functions but in a front-end interface, with no need to perform command line operations in a terminal.

I was able to implement algorithms and data structures by designing an AVL tree, which is a self-balancing tree that stores information. The difference between this self-balancing tree and an unbalanced tree like the one that was in the original program, is that because it self-balances, it creates a more streamlined way of saving the courses in memory, and thus it is faster and more efficient to load them to the user.

As I pivoted to a web app, I replaced the AVL tree with storing courses in a database. This change was made because it provided several benefits, like scalability and data persistence. This change demonstrated scalability because the AVL tree could only handle as much data as the memory allowed. By replacing it with a database it allowed for more data to be loaded and be worked on. It had data persistence because it allowed for the course data to still be stored in the user session since it didn’t rely on volatile memory like the CLI app did.

Completing my coursework throughout the program and developing this ePortoflio has given me greater confidence in my abilities as a programmer. It has helped me not only gain technical skills like debugging, research, creating faster algorithms and working with databases, but also soft skills like patience, and the ability to articulate technical concepts in a simpler way. This combination of technical and soft skills prepares me for a career where I can utilize these skills. A career in software or game development, Dev Ops, machine learning, data science, quantitative analysis, or the many other fields that utilize computer science.
